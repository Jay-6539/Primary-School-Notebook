# 部署步骤详解 - 第二步说明

## 第二步：在哪里执行命令？

### 方式一：使用 Windows PowerShell 或命令提示符（推荐）

1. **打开 PowerShell 或命令提示符**
   - 按 `Win + R` 键
   - 输入 `powershell` 或 `cmd`
   - 按回车

2. **切换到项目目录**
   ```powershell
   cd "D:\Cursor Project\Primary School Aiden"
   ```

3. **或者直接在文件夹中打开**
   - 在文件资源管理器中打开项目文件夹：`D:\Cursor Project\Primary School Aiden`
   - 在地址栏输入 `powershell` 或 `cmd` 然后按回车
   - 或者按住 `Shift` 键，右键点击空白处，选择"在此处打开 PowerShell 窗口"

### 方式二：使用 VS Code 或 Cursor 的终端（最简单）

如果你使用的是 Cursor 或 VS Code：

1. **打开终端**
   - 按 `` Ctrl + ` ``（反引号键，通常在 ESC 键下面）
   - 或者点击菜单：**Terminal → New Terminal**

2. **确认当前位置**
   - 终端会自动在项目目录打开
   - 你会看到类似这样的提示：`PS D:\Cursor Project\Primary School Aiden>`

3. **直接执行命令**
   - 在终端中直接输入命令即可

### 方式三：使用 Git Bash（如果安装了 Git）

1. 在项目文件夹中右键
2. 选择 "Git Bash Here"
3. 执行命令

## 需要先安装 Git

如果你看到 "git 不是内部或外部命令" 的错误，需要先安装 Git：

### 安装 Git

1. **下载 Git**
   - 访问：https://git-scm.com/download/win
   - 下载并安装（使用默认设置即可）

2. **验证安装**
   - 重新打开 PowerShell/命令提示符
   - 输入 `git --version`
   - 如果显示版本号，说明安装成功

## 完整的命令步骤

打开终端后，按顺序执行以下命令：

```bash
# 1. 检查是否已经初始化 git（可选）
git status

# 2. 如果还没有初始化，执行这个
git init

# 3. 添加所有文件
git add .

# 4. 提交代码
git commit -m "Initial commit: Aiden's website"

# 5. 重命名分支为 main（如果还没有）
git branch -M main

# 6. 添加远程仓库（替换 YOUR_REPO_NAME 为你的实际仓库名）
git remote add origin https://github.com/Jay-6539/YOUR_REPO_NAME.git

# 7. 推送到 GitHub
git push -u origin main
```

## 注意事项

- 第一次使用 git 可能需要配置用户名和邮箱：
```bash
git config --global user.name "Jay-6539"
git config --global user.email "你的邮箱@example.com"
```

- 推送时需要输入 GitHub 用户名和密码（或 Personal Access Token）

## 如果遇到问题

1. **"git 不是内部或外部命令"**
   - 需要先安装 Git（见上方说明）

2. **"fatal: not a git repository"**
   - 需要先执行 `git init`

3. **推送时要求认证**
   - 需要使用 Personal Access Token 而不是密码
   - 在 GitHub Settings → Developer settings → Personal access tokens → Tokens (classic) 创建
